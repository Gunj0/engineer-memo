# S3

- Amazon Simple Storage Service
  - オブジェクトストレージサービス
  - マネージドサービス
  - データを「バケット」というコンテナに保存する
  - 静的ウェブサイトホスティング、バックアップ、アーカイブ、ビッグデータ分析などに利用される
  - 料金は保存容量、データ転送量に基づいて課金される
  - メリット
    - スケーラブルで柔軟なストレージ容量を提供
      - オブジェクト数やサイズに制限がない
    - 高い耐久性(99.999999999%) と可用性を提供
      - レプリケーション機能を利用して、データを複数のリージョンに自動的に複製できる
    - セキュリティ機能が充実している(アクセス制御、暗号化、監査ログなど)
    - 他サービスと連携しやすい(AWS Lambda、CloudFront、Glacier など)
    - Static Website Hosting機能を利用して、S3バケットを静的ウェブサイトとしてホスティングできる
  - 制約
    - オブジェクトサイズの上限は5TB
    - バケット内にバケットを作成することはできない
    - AWSアカウント1つにつき、バケットの数は100まで(リクエストすれば増やせる)
    - リージョンやバケット名の変更はできない

## ストレージクラス

- 複数のストレージクラスを提供しており、データのアクセス頻度・耐久性要件・コストに応じて選択できる
  - 標準: 頻繁にアクセスされるデータ向け
  - 低頻度アクセス: 頻繁にはアクセスされないが、必要なときにすぐアクセスできるデータ向け
  - アーカイブ: 長期間保存するが、アクセス頻度が非常に低いデータ向け
  - インテリジェントティアリング: アクセスパターンに基づいて自動的に最適なストレージクラスにデータを移動する
  - ワンゾーン低頻度アクセス: 単一のアベイラビリティゾーンに保存される低頻度アクセスデータ向け

## 操作

- オブジェクトのアップロード、ダウンロード、削除、コピーなどの基本的な操作が可能
- ダッシュボードやAPI、CLI、SDK、SFTPを使用して、バケットやオブジェクトの管理が可能
- バージョニング機能を利用して、オブジェクトの複数のバージョンを管理できる
- ライフサイクルポリシーを設定して、オブジェクトの自動移行や削除が可能
- イベント通知機能を利用して、オブジェクトの変更に対してLambda関数やSNSトピックなどに通知を送ることができる
- インベントリ機能を利用して、バケット内のオブジェクトのリストを定期的に生成できる
- S3 Select機能を利用して、オブジェクト内のデータを直接クエリして必要な部分だけを取得できる

## 用語

- バケット: オブジェクトを保存するコンテナ、グローバルに一意な名前が必要
- オブジェクト: バケット内に保存されるデータの単位
- キー: オブジェクトを識別するための一意の名前
- メタデータ: オブジェクトに関連付けられた情報(例: Content-Type、Content-Lengthなど)
- バージョニング: オブジェクトの複数のバージョンを管理する機能
- マルチパートアップロード: 大きなオブジェクトを複数のパートに分割してアップロードする機能(100MB以上のオブジェクトに推奨)

## セキュリティ

- バケットポリシー: バケット全体のアクセス権限を定義するJSON形式のポリシー
- ユーザーポリシー: IAMユーザーやグループに対してアクセス権限を定義するJSON形式のポリシー
- アクセスコントロールリスト(ACL): 他のAWSアカウントに対してバケットやオブジェクトへのアクセス権限を設定するためのリスト

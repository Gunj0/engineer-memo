# VPC

- Virtual Private Cloud
  - AWS内で論理的に分割された仮想ネットワーク
  - IPアドレス範囲、サブネット、ルートテーブル、ネットワークゲートウェイなどを使用して、AWSリソースを安全に配置できる
  - EC2やRDSでは必須
  - S3, DynamoDB, CloudWatch などの一部サービスはVPCには属さない

## サブネット

- VPC内のIPアドレスの範囲を定義するセグメント, CIDR形式で指定する
- 1つのサブネットは1つのルータを持つ, ルータは共有することも可能
  - パブリックサブネット: ルーティングがインターネットゲートウェイを指し、インターネットにアクセスできるサブネット
  - プライベートサブネット: ルーティングがインターネットゲートウェイを指さず、直接インターネットにアクセスできないサブネット

## 用語

- ルータ
  - ルートテーブル: ネットワークトラフィックのルーティングルールを定義するテーブル, サブネットに関連付ける
  - ネットワークACL: サブネット単位のファイアウォール, インバウンド/アウトバウンドを設定
  - セキュリティグループ: VPC単位のファイアウォール, ホワイトリスト方式でインバウンド/アウトバウンドを設定
- ゲートウェイ
  - インターネットゲートウェイ(IGW): VPCをインターネットに接続するためのゲートウェイ
  - NATゲートウェイ: プライベートIPとグローバルIPを変換し、プライベートサブネットをインターネットに接続するために使用
  - 仮想プライベートゲートウェイ(VGW): VPCをVPNやDirect Connectを介してオンプレミスネットワークに接続するためのゲートウェイ
- 接続
  - VPCエンドポイント: VPCと非VPCサービスを接続する
    - ゲートウェイエンドポイント: S3やDynamoDBなどのAWSサービスに接続, ルートテーブルを使用
    - インターフェースエンドポイント(AWS PrivateLink): ENIを使用してAWS外のサービスに接続
  - VPCピアリング: 異なるVPC間での通信を可能にする接続, AWSアカウントやリージョンを跨いで接続可能
  - Transit Gateway: 複数のVPCやオンプレミスネットワークを接続する拠点となるサービス
- ログ
  - VPCフローログ: VPC内のネットワークトラフィックのログを収集する機能

## CIDR表記

- Classless Inter-Domain Routing
  - IPアドレスとサブネットマスクを組み合わせた表記方法
  - 例: 172.31.0.0/24
    - /24: サブネットマスクのビット数 = ネットワーク部の長さ
      - サブネットマスク: 255.255.255.0
      - ネットワーク部: 172.31.0, ホスト部: 0
        - ホスト範囲: 172.31.0.1 - 172.31.0.254
        - ホスト数: 2^(32 - 24) - 5 = 252
          - 5はネットワークアドレス、ブロードキャストアドレス、ルータ用の予約アドレスを考慮して引く
  - オートスケールするならIPアドレスを十分確保するため、/16や/24等で多めに割り当てる
  - 実際のインスタンスにはDHCPでIPアドレスが割り当てられる
- クラス
  - クラスA: /8, クラスB: /16, クラスC: /24

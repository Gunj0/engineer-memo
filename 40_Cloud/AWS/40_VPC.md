# VPC

- Virtual Private Cloud
  - AWS内で論理的に分割された仮想ネットワーク
  - IPアドレス範囲、サブネット、ルートテーブル、ネットワークゲートウェイなどを使用して、AWSリソースを安全に配置できる
  - EC2やRDSでは必須

## 用語

- サブネット: VPC内のIPアドレスの範囲を定義するセグメント, CIDR形式で指定
  - パブリックサブネット: インターネットゲートウェイを介してインターネットアクセスする
  - プライベートサブネット: インターネットにアクセスできないサブネット
- ゲートウェイ
  - インターネットゲートウェイ: VPCをインターネットに接続するためのゲートウェイ
  - NATゲートウェイ: 一時的にプライベートサブネットをインターネット接続する時などに使用
- セキュリティ
  - セキュリティグループ: VPC単位のファイアウォール, インバウンド/アウトバウンドを設定
  - ネットワークACL: サブネット単位のファイアウォール, インバウンド/アウトバウンドを設定
- ルーティング
  - ルートテーブル: ネットワークトラフィックのルーティングルールを定義するテーブル
- 接続
  - VPCエンドポイント: VPCと非VPCサービスを接続する
    - インターフェースエンドポイント: ENIを使用してAWS外のサービスに接続, PrivateLinkを使用
    - ゲートウェイエンドポイント: S3やDynamoDBなどのAWSサービスに接続, ルートテーブルを使用
  - VPCピアリング: 異なるVPC間での通信を可能にする接続
  - Transit Gateway: 複数のVPCやオンプレミスネットワークを接続する拠点となるサービス
- ログ
  - VPCフローログ: VPC内のネットワークトラフィックのログを収集する機能

## CIDR表記

- Classless Inter-Domain Routing
  - IPアドレスとサブネットマスクを組み合わせた表記方法
  - 例: 172.31.0.0/24
    - /24: サブネットマスクのビット数 = ネットワーク部の長さ
      - サブネットマスク: 255.255.255.0
      - ネットワーク部: 172.31.0, ホスト部: 0
        - ホスト範囲: 172.31.0.1 - 172.31.0.254
        - ホスト数: 2^(32 - 24) - 2 = 254
  - オートスケールするならIPアドレスを十分確保するため、/16や/24等で多めに割り当てる
  - 実際のインスタンスにはDHCPでIPアドレスが割り当てられる
- クラス
  - クラスA: /8, クラスB: /16, クラスC: /24
